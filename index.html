Mode 1 ¬∑ Dial in a vibe

This is the ‚ÄúI know what I want‚Äù entry point. You talk to the system in the same language you use with friends: ‚Äúmore energetic but mid-tempo, less talky, not too acoustic‚Äù.

What the user does

  • Adjust 7 sliders: danceability, energy, valence, tempo, acousticness, instrumentalness, speechiness.
  • Pick how much to trust vibe vs popularity with a single Œª slider.
  • Choose playlist style: tightly focused, balanced, or exploratory.

What the system does

  • Map sliders into the same 7D space as the tracks (with tempo scaled to dataset min/max).
  • Standardize with StandardScaler and apply per-feature weights.
  • Compute weighted cosine similarity to every track.
  • Blend vibe similarity with popularity and greedily build a diverse queue.

Mode 2A ¬∑ Start from a song

When you have one specific track in mind, this mode uses it as an anchor. It‚Äôs basically ‚Äúmore like this‚Äù, but grounded in the actual audio features instead of collaborative filtering.

What the user does

  • Searches the Kaggle library by track or artist name.
  • Picks one seed song from the results.
  • Listens to the recommended queue, with the ability to skip or jump.

What the system does

  • Looks up the seed track‚Äôs standardized 7D embedding.
  • Uses that embedding as the target in the same similarity + scoring pipeline.
  • Drops the seed from the results and returns a queue of its closest neighbors.
  • Explains each recommendation via ‚Äúclosest features‚Äù (e.g. similar energy + tempo + instrumentalness).

For low-friction moments, Vibe Roulette is a single button. It doesn‚Äôt guess genres; it picks a persona like ‚ÄúFlow State Focus‚Äù or ‚ÄúNeon City Ride‚Äù based on weekday vs weekend and time of day, then spins a playlist in that region of the vibe space.

  • Chooses how adventurous they feel: predictable, balanced, or chaotic good.
  • Hits ‚ÄúSpin‚Äù and gets a persona (‚ÄúLazy Brunch Breeze‚Äù, ‚ÄúNight Out Pre-Game‚Äù, etc.).

  • Queries the engine for a larger candidate set and samples ‚Äúnow playing‚Äù via a temperature-controlled distribution.

Under the hood

The project is intentionally simple but opinionated: clear separation of core engine, modes, and UI; and a focus on controllable, interpretable recommendations instead of black-box magic.

Architecture: user input (sliders / seed / time) ‚Üí standardization ‚Üí weighted cosine similarity ‚Üí hybrid scoring (Œª) ‚Üí diversity re-ranking ‚Üí final queue.
Radar chart: comparing two vibes (e.g. morning focus vs party) across danceability, energy, valence, tempo, acousticness, instrumentalness, and speechiness. This is the space cosine similarity operates in.

Vibe engine

  • 7D audio feature vectors per track.
  • Standardization with StandardScaler to align tempo and [0,1] features.
  • Per-feature weights (e.g. slightly more weight on energy & valence).
  • Weighted cosine similarity; similarity mapped from [-1,1] to [0,1].
  • Hybrid scoring: Œª ¬∑ vibe_sim + (1‚àíŒª) ¬∑ popularity_norm.

Ranking & diversity

  • Explicit vs non-explicit filtering and optional min popularity filter.
  • Genre filtering (when available) to keep playlists coherent.
  • Greedy diversity-aware re-ranking using cosine similarity in the weighted space.
  • Prevents ‚Äú10 clones of the same track in a row‚Äù while staying on vibe.
